# Plan: E2E Scripts + Declare + README

## Context

The Falcon account deploy flow works end-to-end (keygen → NTT pk → pack → deploy → transfer) but there are no committed scripts to run it. The existing `apps/demo/scripts/deploy-falcon.mjs` is gitignored (`scripts/` in root `.gitignore`). Need:
1. Idempotent devnet E2E test script
2. Sepolia class declaration support
3. README setup sections

## Key Constraints

- `scripts/` is gitignored — use `bin/` instead
- `bin/declare.sh` already exists and handles devnet/sepolia/mainnet
- `snfoundry.toml` only has devnet profile — need sepolia profile
- `accounts.json` is gitignored (credentials)
- Node.js WASM (`--target nodejs`) and browser WASM (`--target web`) overwrite each other in `falcon-rs/pkg/`

## Steps

### 1. Create `bin/e2e-devnet.sh` — Idempotent devnet E2E test

Orchestrator shell script. Safe to run repeatedly.

```
Flow:
1. Check if devnet running on :5050 — start if not (background, wait for ready)
2. Build Node.js WASM to bin/.wasm-nodejs/ (avoids clobbering browser WASM)
3. Declare FalconAccount class via bin/declare.sh devnet (idempotent)
4. Extract class hash from declare output
5. Run bin/e2e-deploy.mjs with CLASS_HASH and WASM path env vars
6. Print PASS/FAIL summary
```

### 2. Create `bin/e2e-deploy.mjs` — Node.js deploy + transfer

Cleaned-up `deploy-falcon.mjs` with env-var configuration:
- `FALCON_WASM_PATH` — path to Node.js WASM package (default: `bin/.wasm-nodejs`)
- `CLASS_HASH` — declared class hash (required)
- `RPC_URL` — devnet RPC (default: `http://localhost:5050/rpc`)
- Uses starknet.js from `apps/demo/node_modules/`
- Same flow: keygen → NTT pk → pack → compute address → mint → deploy → transfer → verify balance
- Exits 0 on success, 1 on failure

### 3. Add Sepolia profile to `snfoundry.toml`

```toml
[sncast.sepolia]
url = "https://api.zan.top/node/v1/starknet/sepolia/..."
account = "sepolia-deployer"
accounts-file = "accounts.json"
```

`bin/declare.sh sepolia` already works — just needs the profile.

### 4. Add Makefile targets

```makefile
e2e-devnet:
	./bin/e2e-devnet.sh

declare-devnet:
	./bin/declare.sh devnet

declare-sepolia:
	./bin/declare.sh sepolia
```

### 5. Update README.md — setup sections

Add after "Quick start":

- **Setup**: asdf install, devnet, WASM build
- **Devnet E2E**: `make e2e-devnet`
- **Sepolia**: import account, declare, deploy via webapp

### 6. Gitignore `bin/.wasm-nodejs/`

One line in `.gitignore` so the Node.js WASM build cache doesn't get committed.
