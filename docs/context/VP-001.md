## Mandatory Constraints Extracted

### From PRD (`docs/specs/falcon-demo-website.md`)
- Next.js 15 App Router and Bun package manager are required.
- Tailwind CSS v4 must use CSS-first config (no `tailwind.config.js`).
- Landing sections must be RSC (zero client JS).
- Interactive sections must be client components loaded with `next/dynamic` and `ssr: false`.
- Effect patterns are mandatory (`Effect.Service`, `Effect.fn`, `Schema.TaggedError`, Effect Atoms).
- Starknet RPC URL must come from `NEXT_PUBLIC_STARKNET_RPC_URL`.
- Theme tokens required: `falcon-primary`, `falcon-accent`, `falcon-success`, `falcon-error`, `falcon-bg`, `falcon-surface`, `falcon-text`, `falcon-muted`.

### From Architecture (`docs/plans/2026-02-23-falcon-demo-website-design.md`)
- App lives at `apps/demo/`.
- Three-layer structure: Next.js app -> Effect services -> runtime (WASM + starknet.js).
- `WasmRuntime` service lazily loads `/wasm/falcon_rs.js`.
- State uses `@effect-atom/atom` + `@effect-atom/atom-react`.
- Dark theme default with optional light mode.

### From Implementation Plan (`docs/plans/2026-02-23-falcon-demo-website-impl.md`, Task 1)
- Task 1 explicitly defines file contents for config/app shell files.
- Dependency install list includes: `next`, `react`, `react-dom`, `effect`, `@effect-atom/atom`, `@effect-atom/atom-react`, `tailwindcss`, `@tailwindcss/postcss`, `starknet`.
- `next.config.mjs` must enable `asyncWebAssembly`.
- `globals.css` must define falcon color tokens.
- `.env.local` should contain mainnet RPC URL.
- Verification step: run `bun run dev` successfully.

### From Conventions (`CLAUDE.md`)
- Bun is the project package manager.
- Keep implementation aligned with existing package architecture and strict tooling conventions.
