# Context for VP-003: FalconService with Effect.fn Methods Wrapping WASM

## Scope
VP-003 validates and hardens the FalconService service layer used by the verification playground. The service already existed; this ticket adds missing tests, verifies architecture constraints, and records implementation context.

## FalconService Architecture

File: `apps/demo/src/services/FalconService.ts`

- Service pattern: `Effect.Service<FalconService>()("FalconService", { accessors: true, effect: ... })`
- Dependency injection: `yield* WasmRuntime` (via Layer composition in callers/tests)
- Tracing: every method wrapped with `Effect.fn("Falcon.<method>")`
- Error handling: all fallible WASM calls wrapped in `Effect.try`
- Error types: `KeygenError`, `SigningError`, `VerificationError`, `HintGenerationError`, `PackingError` (all `Schema.TaggedError`)
- No forbidden patterns in FalconService:
  - no `Generator<any>` annotations
  - no `Option.getOrThrow`
  - no raw `try/catch` blocks in `Effect.gen`

## Methods

- `generateKeypair(seed?)`
  - Calls `wasm.keygen`
  - Deserializes `vk` bytes to 512 NTT coefficients with `parsePublicKeyNttBytes`
  - Returns `FalconKeypair`
- `sign(secretKey, message)`
  - Calls `wasm.sign(secretKey, message, new Uint8Array(0))`
  - Returns `FalconSignatureResult`
- `verify(verifyingKey, message, signature)`
  - Calls `wasm.verify(...)`
  - Returns `boolean`
- `createHint(s1, pkNtt)`
  - Calls `wasm.create_verification_hint(...)`
  - Returns `Uint16Array`
- `packPublicKey(pkNtt)`
  - Calls `wasm.pack_public_key_wasm(...)`
  - Returns `PackedPublicKey` with `slots`
- `deserializePublicKeyNtt(vkBytes)`
  - Exposes standalone VK deserialization
  - Returns `Int32Array`

## VK Deserialization Notes

`parsePublicKeyNttBytes` parses Falcon-512 VK format:
- byte 0: header/tag
- bytes 1-896: 512 coefficients packed in 14-bit LSB-first format

Validation enforced:
- input length must be at least 897 bytes
- each extracted coefficient must be `< Q` where `Q = 12289`

## Test Hardening Added in VP-003

File: `apps/demo/src/__tests__/verify-playground/falcon-service.test.ts`

New coverage added:
1. `deserializePublicKeyNtt` fails with `KeygenError` for out-of-range coefficients (`0xFF` packed data)
2. Error message propagation checks for:
   - `KeygenError`
   - `SigningError`
   - `VerificationError` (`step: "verify"` retained)
3. Full pipeline integration checks:
   - `keygen -> sign -> verify -> true`
   - `keygen -> sign -> verify -> false`
4. `packPublicKey` slot format validation (`^0x[0-9a-fA-F]+$`)

Result: FalconService test file now runs **26 pass / 0 fail**.
