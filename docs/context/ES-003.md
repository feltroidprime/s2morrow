  - Details the P0 requirement for `effect-services`: `WasmRuntime` must load falcon-rs WASM lazily from `/wasm/` via fetch+instantiate, cache the module, and fail with `WasmLoadError`.

  - Contains the original planned implementation of `WasmRuntime` (Task 3, Step 4) which used an unsafe `let cachedModule: WasmModule | null = null` state and an `as unknown as WasmModule` assertion.

- `CLAUDE.md`
  - Project conventions, emphasizing strict adherence to Effect-TS patterns (Effect.Service, Context.Tag, Schema.TaggedError).

## Goals for ES-003
1. **Create `apps/demo/src/services/WasmRuntime.ts`**.
2. **Context.Tag**: Define `WasmRuntime` using the Effect `Context.Tag` pattern.
3. **Runtime Type Validation**: Avoid blind type casting. Implement a runtime check (e.g., `typeof mod.keygen === 'function'`) for every expected method in `WasmModule`.
4. **Effect-native Caching**: Replace the mutable `let cachedModule = null` with Effect state primitives (like `Ref`) or rely on Effect Layer memoization/Option to manage the WASM loading state properly.
5. **WasmLoadError**: Use `Schema.TaggedError` for the loading failure.
6. **Tests**: Include a test file that demonstrates injecting a mock WASM module using `Layer.succeed(WasmRuntime, mockModule)`.
