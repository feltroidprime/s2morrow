## How VK Bytes Encode the Public Key (from falcon-rs)

### Encoding Format

The verifying key (`vk`) is 896 bytes containing 512 coefficients, each stored as 14 bits packed LSB-first:

```
VK bytes (896): [14-bit coef_0][14-bit coef_1]...[14-bit coef_511]
Total: 512 × 14 bits = 7168 bits = 896 bytes
```

### Rust Implementation (falcon-rs/src/encoding.rs)

**Serialization** (`serialize_public_key`):
- Each coefficient stored as 14 bits, packed LSB-first within bytes
- 512 coefficients × 14 bits = 7168 bits = 896 bytes

**Deserialization** (`deserialize_public_key`):
- Reverses the bit unpacking
- Returns `[i32; 512]` array
- Validates all coefficients are in `[0, Q)` where Q = 12289

### TypeScript Deserialization Algorithm

To deserialize VK bytes into an Int32Array of 512 coefficients:

```typescript
function deserializePublicKeyNtt(vkBytes: Uint8Array): Int32Array {
  const Q = 12289
  const N = 512
  const BITS_PER_COEFF = 14
  const coeffs = new Int32Array(N)

  let bitBuffer = 0
  let bitsInBuffer = 0
  let byteIndex = 0
  const mask = (1 << BITS_PER_COEFF) - 1 // 0x3FFF

  for (let i = 0; i < N; i++) {
    while (bitsInBuffer < BITS_PER_COEFF) {
      bitBuffer |= vkBytes[byteIndex++] << bitsInBuffer
      bitsInBuffer += 8
    }
    const coeff = bitBuffer & mask
    bitBuffer >>= BITS_PER_COEFF
    bitsInBuffer -= BITS_PER_COEFF

    if (coeff >= Q) {
      throw new Error(`Invalid coefficient ${coeff} at index ${i} (must be < ${Q})`)
    }
    coeffs[i] = coeff
  }

  return coeffs
}
```

### Key Insight: h polynomial IS the NTT-domain public key

In falcon-rs, the `VerifyingKey` struct contains `h: [i32; N]` which is the public polynomial `h = g/f mod (x^n + 1, q)`. This `h` is algebraically in the "NTT-compatible" form — it's what gets passed directly to `generate_mul_hint` as `pk_h_ntt` and to `pack_public_key` for Starknet storage.

So the deserialized coefficients from VK bytes ARE the `publicKeyNtt` values needed for hint generation and packing.

### Test Mock Updates

The mock `makeSuccessWasm` currently returns:
```typescript
keygen: (_seed) => ({
  sk: new Uint8Array(1281).fill(1),
  vk: new Uint8Array(897).fill(2),
})
```

This mock VK (`new Uint8Array(897).fill(2)`) will deserialize to specific coefficient values via 14-bit unpacking. The test should either:
1. Use a realistic VK mock that deserializes to known values, OR
2. Create a VK from known coefficients using a `serializePublicKey` helper, OR
3. Simply verify the deserialized values are non-zero and within `[0, Q)`

**Note**: `Uint8Array(897).fill(2)` unpacks as: byte pattern `0x02 0x02 0x02...` → bits `00000010 00000010 00000010...` → 14-bit groups: `0b00000010000000` = 128 for first, then shifts. The exact values depend on bit packing order but they will be consistent and deterministic.
