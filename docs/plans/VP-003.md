**Goal:** Verify, test-harden, and document the FalconService Effect service that wraps falcon-rs WASM with typed errors and OpenTelemetry span tracing.

**Architecture:** FalconService is an `Effect.Service` with `accessors: true` that depends on `WasmRuntime` (a `Context.Tag<WasmModule>`). Each method uses `Effect.fn()` for automatic span tracing and `Effect.try()` for synchronous error capture with `Schema.TaggedError` types. No `Generator<any>` annotations, no `Option.getOrThrow`, no raw try-catch — pure Effect error handling.

**Tech Stack:** Effect-TS (`Effect.Service`, `Effect.fn`, `Schema.TaggedError`, `Context.Tag`), Bun test runner, TypeScript strict mode

// ---------------------------------------------------------------------------
// Integration: full pipeline (keygen → sign → verify)

describe("FalconService full pipeline integration", () => {
  it("keygen → sign → verify → true through service layer", async () => {
    // Mock that returns self-consistent data: keygen produces a keypair,
    // sign produces a signature, verify returns true for any input.
    // This tests that data flows correctly between service methods.
    const mockWasm: WasmModule = {
      ...makeSuccessWasm(),
      keygen: (_seed) => ({
        sk: new Uint8Array(1281).fill(7),
        // Valid VK: header + all-zero data → coefficients all 0 (< Q=12289)
        vk: new Uint8Array(897),
      }),
      sign: (_sk, _msg, _salt) => ({
        signature: new Uint8Array(666).fill(9),
        salt: new Uint8Array(40).fill(10),
      }),
      verify: (_vk, _msg, _sig) => true,
    }
    const layer = Layer.succeed(WasmRuntime, mockWasm)

    // Step 1: Generate keypair
    const keypairExit = await runWith(
      FalconService.generateKeypair(new Uint8Array(32)),
      layer,
    )
    expect(Exit.isSuccess(keypairExit)).toBe(true)
    if (!Exit.isSuccess(keypairExit)) return

    const keypair = keypairExit.value
    expect(keypair.secretKey.length).toBe(1281)
    expect(keypair.verifyingKey.length).toBe(897)
    expect(keypair.publicKeyNtt.length).toBe(512)

    // Step 2: Sign a message
    const message = new TextEncoder().encode("Post-quantum on Starknet!")
    const signExit = await runWith(
      FalconService.sign(keypair.secretKey, message),
      layer,
    )
    expect(Exit.isSuccess(signExit)).toBe(true)
    if (!Exit.isSuccess(signExit)) return

    const sigResult = signExit.value
    expect(sigResult.signature.length).toBe(666)
    expect(sigResult.salt.length).toBe(40)

    // Step 3: Verify the signature
    const verifyExit = await runWith(
      FalconService.verify(keypair.verifyingKey, message, sigResult.signature),
      layer,
    )
    expect(Exit.isSuccess(verifyExit)).toBe(true)
    if (!Exit.isSuccess(verifyExit)) return
    expect(verifyExit.value).toBe(true)
  })

  it("keygen → sign → verify → false (tampered signature)", async () => {
    let callCount = 0
    const mockWasm: WasmModule = {
      ...makeSuccessWasm(),
      keygen: (_seed) => ({
        sk: new Uint8Array(1281).fill(7),
        vk: new Uint8Array(897),
      }),
      sign: (_sk, _msg, _salt) => ({
        signature: new Uint8Array(666).fill(9),
        salt: new Uint8Array(40).fill(10),
      }),
      verify: (_vk, _msg, _sig) => {
        callCount++
        return false // simulate invalid/tampered signature
      },
    }
    const layer = Layer.succeed(WasmRuntime, mockWasm)

    const keypairExit = await runWith(
      FalconService.generateKeypair(new Uint8Array(32)),
      layer,
    )
    expect(Exit.isSuccess(keypairExit)).toBe(true)
    if (!Exit.isSuccess(keypairExit)) return

    const signExit = await runWith(
      FalconService.sign(keypairExit.value.secretKey, new TextEncoder().encode("msg")),
      layer,
    )
    expect(Exit.isSuccess(signExit)).toBe(true)
    if (!Exit.isSuccess(signExit)) return

    const verifyExit = await runWith(
      FalconService.verify(
        keypairExit.value.verifyingKey,
        new TextEncoder().encode("msg"),
        signExit.value.signature,
      ),
      layer,
    )
    expect(Exit.isSuccess(verifyExit)).toBe(true)
    if (!Exit.isSuccess(verifyExit)) return
    expect(verifyExit.value).toBe(false)
    expect(callCount).toBe(1)
  })
})
```

### WasmModule Interface Contract

```typescript
interface WasmModule {
  keygen(seed: Uint8Array): { sk: Uint8Array; vk: Uint8Array }
  sign(skBytes: Uint8Array, message: Uint8Array, salt: Uint8Array): { signature: Uint8Array; salt: Uint8Array }
  verify(vkBytes: Uint8Array, message: Uint8Array, signature: Uint8Array): boolean
  create_verification_hint(s1: Int32Array, pkNtt: Int32Array): Uint16Array
  pack_public_key_wasm(pkNtt: Uint16Array): string[]
  public_key_length(): number  // Returns 897
  salt_length(): number        // Returns 40
}
```

### VK Deserialization (`parsePublicKeyNttBytes`)
- Falcon-512 VK = 897 bytes: 1-byte header + 896 bytes (512 × 14-bit coefficients, LSB-first)
- Bit-buffer window extraction algorithm
- Validates: minimum length, all coefficients < Q=12289

### Architecture Decisions
1. **No `dependencies` array** — FalconService doesn't use `dependencies: [WasmRuntimeLive]` from the plan.
   Instead, callers compose layers explicitly. This prevents the live WASM loader from firing at import
   time, which would fail in test environments.
2. **`Effect.try` not `Effect.tryPromise`** — All WASM calls are synchronous (WASM functions are sync),
   so `Effect.try` is correct. Only WasmRuntime's `loadWasmModule` uses `Effect.tryPromise` (for dynamic import + init).
3. **`satisfies` for return types** — Methods use `satisfies FalconKeypair` etc. for compile-time checking
   without narrowing the inferred type, which preserves Effect's type inference.
4. **Bonus method `deserializePublicKeyNtt`** — Not in original ticket spec but needed for account-deploy
   flow (re-deserialize a saved VK without re-generating keypair).

### Security Invariants
- Secret keys never leave the browser — only passed to WASM `sign()` as `Uint8Array`
- No server-side crypto operations — FalconService is client-only
- No hardcoded private keys — seed from `crypto.getRandomValues()` or caller-provided
