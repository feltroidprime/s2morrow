# Plan: VP-001 - Scaffold Next.js 15 app at `apps/demo/` with Tailwind v4 and Effect-TS

## Overview
VP-001 is foundational scaffolding for all downstream demo work. The codebase already contains most required files, so this plan is regression-focused: add missing test coverage first, then apply minimal implementation changes (primarily `.env.local`) until all scaffold acceptance checks pass.

This plan enforces:
- Next.js 15 App Router under `apps/demo/`
- Tailwind v4 CSS-first setup (no `tailwind.config.js`)
- Required runtime dependencies (`effect`, Effect Atoms, `starknet`)
- No server-side crypto or secret material in scaffold config
- Bun-based verification (`bun test`, `bun run dev`)

## Risks and Mitigations
1. `.env.local` may be gitignored and absent in fresh clones.
- Mitigation: keep test scoped to local dev expectations; add `.env.example` follow-up ticket if reproducibility is required.

2. Existing tracked `.next` artifacts can cause noisy diffs and unstable status.
- Mitigation: restrict VP-001 changes to scaffold files/tests only; avoid touching build outputs.

3. Public RPC endpoint instability can fail runtime smoke checks.
- Mitigation: treat `bun run dev` startup success as acceptance; avoid network-dependent RPC calls in VP-001.

4. Regression risk from broad edits to already-correct scaffold.
- Mitigation: test-first, then minimal patching only where failures occur.
