# Plan: ES-004 - Implement FalconService with correct `publicKeyNtt` deserialization

## Overview

The service remains an `Effect.Service` with `accessors: true` and `dependencies: [WasmRuntimeLive]`, keeps `Schema.TaggedError` failures, and removes explicit `Generator<any, *, any>` annotations so `Effect.fn` infers method types.

Implementation will follow strict TDD: tests first, then service changes.

## Risks and Mitigations
1. Risk: Verifying-key packing format mismatch (bit order/off-by-one) produces wrong coefficients.
- Mitigation: Use deterministic pack/unpack fixture tests with boundary coefficients (`0`, `1`, `16383`) and multiple bit-offset alignments.

2. Risk: Adding `dependencies: [WasmRuntimeLive]` makes tests accidentally try loading real browser WASM.
- Mitigation: In tests, explicitly provide a mock `WasmRuntime` layer to override runtime dependency for every effect under test.

3. Risk: Failure-path assertions become brittle if checking serialized cause strings.
- Mitigation: Assert typed error tags/fields (`_tag`, `step`) rather than only `JSON.stringify(cause)`.

4. Risk: Browser-only `crypto.getRandomValues` behavior in test runtime.
- Mitigation: Keep explicit-seed tests as primary path; for no-seed path, stub `globalThis.crypto` deterministically when needed.
