## Risks and Mitigations
1. Risk: `.env.local` is gitignored and easy to miss in CI/local onboarding.
- Mitigation: keep explicit unit test that checks presence and key format; document fallback env var in onboarding notes if needed.

2. Risk: `next-env.d.ts` may not exist before first `bun run dev`.
- Mitigation: include it in `tsconfig` per Next.js standard and run dev smoke once in verification to generate it.

3. Risk: dev smoke script can leave orphan Next.js processes.
- Mitigation: enforce `trap` cleanup and process-group kill in script.

4. Risk: future dependency upgrades accidentally remove required Effect libraries.
- Mitigation: dependency contract unit test blocks regressions early.
